# 盒打击应用重构计划

## 1. 项目现状分析

### 1.1 当前技术栈
- **前端**：React 18 + TypeScript + Tailwind CSS
- **数据可视化**：Chart.js + react-chartjs-2
- **数据存储**：SQLite3 (Node.js环境) / 内存存储 (浏览器环境)
- **系统监控**：Node.js child_process 执行 Windows 命令

### 1.2 现有功能
- 应用使用统计（饼图和柱状图）
- 资源占用分析（CPU和内存）
- 时间轴统计（折线图）
- 异常应用检测
- 最近进程活动日志显示

### 1.3 主要文件结构
- `src/pages/`：应用页面组件
- `src/components/`：可复用组件
- `src/utils/`：工具函数（监控、分析、数据库）
- `src/App.tsx`：应用主组件

## 2. 重构目标

### 2.1 技术目标
- 转换为可直接下载的桌面应用（.exe文件）
- 实现数据持久化存储
- 保持现有功能完整性
- 优化用户体验

### 2.2 非技术目标
- 简化用户安装和使用流程
- 提供更稳定的系统监控能力
- 确保数据安全和隐私保护

## 3. 技术方案

### 3.1 桌面应用框架选择

**候选方案**：
1. **Electron**
   - 优势：跨平台支持，丰富的API，成熟的生态系统
   - 劣势：包体积较大，内存占用较高

2. **Tauri**
   - 优势：轻量级，启动速度快，包体积小
   - 劣势：生态系统相对较新，某些功能可能不够成熟

**推荐方案**：Electron
- 理由：Electron是最成熟的桌面应用框架，拥有丰富的社区资源和插件生态，能够更好地满足Windows系统监控的需求
- 关键依赖：
  - `electron`：核心框架
  - `electron-builder`：用于打包生成.exe文件
  - `electron-updater`：用于应用自动更新

### 3.2 数据持久化方案

**候选方案**：
1. **SQLite3**
   - 优势：轻量级，无需额外服务，适合本地应用
   - 劣势：并发性能有限

2. **LevelDB**
   - 优势：高性能，适合键值存储
   - 劣势：查询功能相对简单

**推荐方案**：SQLite3
- 理由：与现有实现兼容，适合结构化数据存储，支持复杂查询
- 关键依赖：
  - `sqlite3`：Node.js SQLite3驱动
  - `better-sqlite3`：性能优化的SQLite3封装

### 3.3 系统监控方案

**推荐方案**：
- 保留现有的Windows命令执行方式
- 增加错误处理和重试机制
- 优化命令执行性能
- 关键依赖：
  - `child_process`：Node.js内置模块
  - `windows-process-list`：Windows进程信息获取
